WebMUGEN Design Philosophy

長期開発、MUGEN互換、通信対戦、AI共同開発を見据えた設計思想。

最重要思想

  1. WebMUGEN本体はサーバーに依存しない。 ゲーム実行はブラウザ内で完結させる。
  2. core はMUGENバージョンを知らない。 WinMUGENやMUGEN 1.0の差はparserとcompatibility layerで吸収する。
  3. 描画とゲームロジックを分離する。 rendererはGameStateを描画するだけで書き換えない。
  4. rollback通信対戦を最初から意識する。 入力はフレーム単位、状態は保存・復元・再計算可能にする。
  5. 調査したMUGEN仕様は必ずHTMLドキュメントに残す。 チャットの中だけに残さない。

避けること

  • Canvas描画中に座標や体力を更新する。
  • WinMUGEN固有の癖をcoreへ直接埋め込む。
  • ゲームロジックを現在時刻や非決定的乱数に依存させる。
  • PHPや特定レンタルサーバーにWebMUGEN本体を依存させる。
