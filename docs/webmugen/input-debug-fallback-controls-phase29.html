Input Debug and Fallback Controls Phase29

This phase adds reusable input debug helpers and fallback control logic.

  • Shows which keys are pressed
  • Maps P1/P2 inputs to debug flags
  • Provides fallback player state changes for movement, jump, attack, and projectile

Integration target: call createInputDebugSnapshot() from the app loop, render formatInputDebugOverlay(), and apply applyFallbackControls() before/after CNS execution when CNS does not change state.
