TargetBind

WinMUGEN互換仕様・WebMUGEN実装監査（2026-07-25）

概要

登録済みTargetを、Controller実行者の軸を基準とする相対位置へ一定時間拘束する。主用途は投げ・掴み・壁運搬である。

根拠の区分

区分確認内容
Elecbyte公式TargetBind は指定Targetを自軸相対に拘束する。time 既定値は1、ID 既定値は-1（全Target）、pos 既定値は0,0。速度副作用は明記されていない。
WinMUGEN公式文書ミラー2002-04-14版でも同じ既定値と相対位置仕様を確認。
互換実装（補助根拠）IKEMEN GO は有効なTarget bind中、拘束元の速度を対象へ同期し、TargetDrop時にbindを解除する。これは公式文書にない挙動の補助根拠であり、WinMUGEN実機結果とは区別する。
WinMUGEN実機この監査環境では実行可能な mugen.exe を確認できず、未検証。したがってCompatibility MatrixはPartialのままとする。

パラメータと時間

項目WebMUGENの扱い
ID指定時は一致するHitDef IDの登録Target、未指定時は全登録Targetを選ぶ。
pos = x,yXは拘束元Facing相対、Yはワールド軸。省略は0,0。
time省略1 tick。
time = 0位置を変更せず、既存の同一所有者bindを解除。
time > 0指定tick数。実行tickを含み、各可動tick後に減算。
time < 0内部値-1へ正規化し無期限。公式文書外のため実機確認は残る。

位置・速度の更新モデル

  1. Controller実行時に対象を owner.x + pos.x * owner.facing, owner.y + pos.y へ置き、ownerの現在のX/Yワールド速度をコピーする。
  2. 両root playerのphysics後、同じ位置拘束と速度同期を再適用する。
  3. 画面端clamp・PlayerPush後に最終補正し、描画時の1フレームずれを防ぐ。
  4. 期限切れではbind情報だけを消す。拘束前速度へ戻さず、0固定もせず、最後に同期した速度を保持する。

解除・再実行・停止時間

  • 同じ対象への再実行は位置・速度・残り時間を新しい値で上書きする。
  • TargetDrop は対象登録と、そのownerによるbindを同時に解除する。
  • TargetState/SelfState 自体では解除しない。投げ先State遷移をまたぐ継続を許す。
  • どちらかの参加者がHitPause中なら有限時間を消費しない。通常Pause/SuperPauseでは既存のentity可動判定とphysics復元を利用するが、個別movetime組合せは未検証。

既知の制限・残作業

  • WinMUGEN実機で、速度同期・解除tick・負値・Pause/SuperPauseを確認する。
  • Helper/team/multi-player Targetと複数localcoord間のスケーリングは未対応。
  • 実装はroot playerの共通runtimeにあり、KFM/特定State番号への特例はない。
  • 焦点テストは静止/移動元、Facing、既定/0/有限/負時間、HitPause、TargetDrop、画面端、T-H-M-A State 3430をカバーする。
